# Dismai 并发性能测试报告

## 测试环境
- CPU: 64核 | 内存: 503GB | Docker Compose 单节点
- 工具: wrk (williamyeh/wrk Docker) + Python requests
- 测试节目: 1500座, 1票档
- 日期: 2026-05-24

---

## 演进历史

### v1: 基准（1实例, 无分片, 无快速拒绝）
| 连接数 | 用户池 | TPS | 超时 | 备注 |
|--------|--------|-----|------|------|
| 50 | 455 | 74 | ~430 | 单票档串行锁瓶颈 |

### v2: 库存分片（1实例, 10分片, 无快速拒绝）
| 连接数 | 用户池 | TPS | 超时 | 备注 |

| 50 | 455 | 81 | 2243 | 分片生效但单JVM本地锁仍是瓶颈 |

## 总体提升

| 指标 | v1 | v4 | 提升 |
|------|-----|-----|------|
| 峰值 TPS | 74 | 269 | 3.6× |
| 500连接超时 | ~430 | 0 | 消除 |
| 不超卖 | ✓ | ✓ | ✓ |

## 架构

```
Nginx → Gateway → N × program-service (各10分片本地锁)
                      ↓ Redis (分布式锁 + Lua原子 + total_remain快速拒绝)
```
