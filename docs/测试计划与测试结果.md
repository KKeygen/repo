# Dismai 项目测试计划与测试结果

## 1. 测试目标

本测试文档用于支撑期末项目 demo 与答辩，目标是确认项目在展示前不存在明显阻塞性回归：后端核心模块可以编译，前端可以构建，公共契约和自动选座等关键逻辑有可重复运行的单元测试覆盖。

## 2. 测试范围

| 类型 | 范围 | 当前状态 |
| --- | --- | --- |
| 后端构建测试 | Maven 多模块编译，使用 JDK 17 | 已执行 |
| 前端构建测试 | Vue 3/Vite 生产构建 | 已执行 |
| 单元测试 | 公共响应结构、业务码解析、订单版本解析、字符串工具、自动选座算法 | 已补充并执行 |
| 集成测试 | MySQL、Redis、Kafka、Nacos、Elasticsearch 依赖链路 | 待补充 |
| 压力测试 | 防刷、锁座、下单、支付回调等高并发场景 | 计划与结果待补充 |

## 3. 测试环境

| 项目 | 说明 |
| --- | --- |
| Java | JDK 17 |
| Maven | 3.9.x |
| Node.js | 适配 Vue 3/Vite 构建 |
| 容器 | Docker，可使用 `maven:3.9-eclipse-temurin-17` 镜像规避本机 JDK 版本不一致问题 |
| 外部依赖 | MySQL、Redis、Kafka、Nacos、Elasticsearch 在集成/压测阶段准备 |
